  - PRD：`.trae/documents/3D打印产品展示与管理平台-PRD.md`
  - 技术架构：`.trae/documents/3D打印产品展示与管理平台-技术架构.md`

## 背景与目标
- 背景：询价需要可解释的“预估报价区间”，同时站点内容（导航、首页区块、联系信息等）需要可配置，降低每次改动都改代码的成本。
- 目标用户/使用场景：管理员维护报价规则与站点基础设置；访客获取预估报价并提交询价。

## 范围
- In scope：
  - 报价规则管理：
    - 规则维度：材料/工艺/精度/数量/交期等加价系数与阶梯规则。
    - 规则版本与启用状态（enabled/version）。
    - 估价接口：`POST /api/quote/estimate` 按启用规则输出 `priceMin/priceMax/disclaimer`。
  - 站点设置：
    - 导航配置、首页区块开关与排序（可逐步实现）。
    - 联系信息、社媒链接、公告（可选）。
- Out of scope：
  - 复杂规则编排器与回测系统（可后续增强）。

## 用户故事
- 作为管理员，我想配置一套可维护的报价规则，并调整站点基础信息，从而让询价预估与站点展示能快速迭代。

## 验收标准（必须可测试）
- [ ] 管理员登录后可访问 `/admin/pricing` 与 `/admin/settings`，未登录不可访问。
- [ ] 报价规则支持创建/编辑/启用/停用（最少保证“当前启用规则”可查询）。
- [ ] `POST /api/quote/estimate` 返回的 `disclaimer` 明确标注“仅供参考”，并带规则版本（如已实现）。
- [ ] 站点设置修改后在前台可见（如果当前版本还未接入设置，则至少保留接口与数据结构占位）。

## 边界与反例用例
- 无启用规则：估价接口返回默认规则或明确错误（策略在 Tech Spec 定稿；建议返回默认区间+提示）。
- 规则不合法：阶梯区间重叠、负数价格、min > max 时拒绝保存并提示。
- 兼容：规则版本升级后，历史询价的 estimate 保留原始版本信息（如需求需要）。

## 依赖与约束
- 相关入口：
  - 前端页面：`workspace/src/pages/admin/Pricing.tsx`、`workspace/src/pages/admin/Settings.tsx`
  - 后端路由：`workspace/api/routes/admin/pricing.ts`
  - 估价逻辑：`workspace/api/pricing.ts`、`workspace/api/routes/quote.ts`
  - 数据表：`pricing_rule`

## 实现拆分（最小可回滚）
- 子任务 1：报价规则存储（DB）与读取“当前启用规则”。
- 子任务 2：估价接口按规则计算区间并返回免责声明与版本。
- 子任务 3：后台规则管理 UI（列表/编辑/启用）。
- 子任务 4：站点设置存储与前台接入（逐步实现）。

## 验证方式
- 自动化：`npm run check`、`npm run lint`、`npm run build`
- 手动路径：
  - 后台创建/启用规则 → 前台询价页点击预估报价 → 返回区间与免责声明
  - 输入非法规则数据应被拒绝

## 发布与回滚
- 发布检查点：估价结果合理、规则启用切换可控、前台不因规则异常而不可用。
- 回滚方式：发布前备份 SQLite；规则变更支持回退到上一版本规则记录并重新启用。
