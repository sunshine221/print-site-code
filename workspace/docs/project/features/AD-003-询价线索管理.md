# 需求卡片：AD-003 询价线索管理

## 基本信息
- 编号：AD-003
- 名称：询价线索管理（状态流转/备注/导出）
- 状态：draft
- 关联文档：
  - PRD：`.trae/documents/3D打印产品展示与管理平台-PRD.md`
  - 技术架构：`.trae/documents/3D打印产品展示与管理平台-技术架构.md`

## 背景与目标
- 背景：询价提交后需要可追踪、可跟进的线索管理，形成从“新建”到“已报价/已关闭”的闭环。
- 目标用户/使用场景：管理员查看新询价→补充备注→状态流转→必要时导出 CSV 做线下协作。

## 范围
- In scope：
  - 线索列表：按创建时间排序，支持基础筛选（状态/关键字）与分页（可逐步实现）。
  - 线索详情：展示访客提交内容、附件、预估报价信息（如有）。
  - 状态流转：`new → processing → quoted / closed`。
  - 备注：内部备注（internalNote）可编辑保存。
  - 导出：CSV 导出（可选，先占位后实现）。
- Out of scope：
  - 多人分配、复杂跟进记录体系（可后续增强）。

## 用户故事
- 作为管理员，我想在后台集中查看与更新询价线索状态，并记录跟进备注，从而提高响应效率与转化率。

## 验收标准（必须可测试）
- [ ] 管理员登录后可访问 `/admin/inquiries`，未登录不可访问。
- [ ] 列表能展示至少：id、创建时间、状态、核心联系人信息（name/email/phone）。
- [ ] 进入详情后可更新状态与内部备注，保存成功后列表状态同步更新。
- [ ] 状态流转仅允许在约定枚举内，非法状态写入被拒绝并提示错误。
- [ ] 导出入口存在（若未实现导出，需明确提示“暂未开放/开发中”）。

## 边界与反例用例
- 并发：同一线索被同时更新时的覆盖策略（默认最后写入覆盖，后续可扩展乐观锁）。
- 附件：附件不可访问/已被删除时的展示策略（显示不可用提示）。
- 大数据量：列表需分页或限制条数，避免一次性渲染过多。

## 依赖与约束
- 相关入口：
  - 前端页面：`workspace/src/pages/admin/Inquiries.tsx`
  - 后端路由：`workspace/api/routes/admin/inquiries.ts`
  - 数据表：`inquiry`、`inquiry_attachment`、`media`

## 实现拆分（最小可回滚）
- 子任务 1：列表 + 详情展示（只读）。
- 子任务 2：状态更新 + 内部备注（写入）。
- 子任务 3：筛选/分页（增强）。
- 子任务 4：CSV 导出（可选）。

## 验证方式
- 自动化：`npm run check`、`npm run lint`、`npm run build`
- 手动路径：
  - 公开端提交一条询价（FE-004）→ 后台列表可见
  - 打开详情 → 更新状态与备注 → 刷新后仍保留
  - 非法状态更新应被拒绝

## 发布与回滚
- 发布检查点：线索列表可用、状态流转正确、备注可保存、鉴权生效。
- 回滚方式：若涉及数据库迁移，发布前备份 SQLite；回滚时恢复备份并回滚后端版本。
