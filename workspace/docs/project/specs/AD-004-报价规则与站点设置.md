# Tech Spec：AD-004 报价规则与站点设置

## 1. 目标与非目标
- 目标：报价规则可版本化维护并驱动估价接口；站点基础设置可配置并逐步接入前台展示。
- 非目标：复杂规则编排器、规则回测系统。

## 2. 现状说明（基于代码）
- 相关入口：
  - 前端：`workspace/src/pages/admin/Pricing.tsx`、`workspace/src/pages/admin/Settings.tsx`
  - 后端：`workspace/api/pricing.ts`、`workspace/api/routes/quote.ts`、`workspace/api/routes/admin/pricing.ts`
  - 数据表：`pricing_rule`
- 现有行为：估价接口与后台页面已存在（以当前实现为准）。
- 现有问题：待补齐（规则编辑体验、规则启用切换策略、站点设置落库与前台接入）。

## 3. 设计概览
- 报价规则：
  - 规则存储为 JSON（rule_json），并提供 version/enabled 字段。
  - 估价优先读取 enabled=1 的最新规则；无启用规则时使用默认值并给出提示。
- 站点设置：
  - 初期可采用单表（site_setting）或复用 JSON 配置（待定稿）。

## 4. API 契约（如涉及）
- `POST /api/quote/estimate`：返回 `ruleVersion` 与 `disclaimer`
- admin 报价规则：
  - `GET /api/admin/pricing/rules`
  - `POST /api/admin/pricing/rules`
  - `PATCH /api/admin/pricing/rules/:id`
  - `POST /api/admin/pricing/rules/:id/enable`
- 站点设置（占位）：
  - `GET /api/admin/settings`
  - `PATCH /api/admin/settings`

## 5. 测试计划
- 手动回归路径：
  - 启用规则 → 询价页估价返回区间与免责声明
  - 禁用所有规则 → 估价降级策略符合预期
  - 规则非法输入被拒绝

## 6. 发布与回滚
- 发布：先备份 SQLite；再发布后端与前端；如规则变更涉及数据迁移则先执行迁移。
- 回滚：恢复备份并回滚后端版本；重新启用上一版本规则记录。
