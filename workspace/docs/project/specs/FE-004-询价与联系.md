# Tech Spec：FE-004 询价/联系（双路径）

## 1. 目标与非目标
- 目标：双路径询价（service/product）可用，具备估价、提交、回执，并可为后台线索管理提供数据闭环。
- 非目标：邮件回执、模型体积/尺寸自动解析、复杂幂等键体系（可后续增强）。

## 2. 现状说明（基于代码）
- 相关入口：
  - 页面：`workspace/src/pages/Quote.tsx`
  - API：`POST /api/quote/estimate`、`POST /api/inquiries`
- 现有行为：页面已支持双路径参数 `type`，并实现估价与提交。
- 现有问题：待补齐（输入校验强度、估价失败策略、提交幂等与错误码一致性）。

## 3. 设计概览
- 方案摘要：
  - 双路径参数：`type=service|product`，product 需 `productId` 与 `skuId`。
  - 估价：优先保证返回结构稳定，规则版本与免责声明一致。
  - 提交：服务路径必须有附件；产品路径必须有 skuId。
- 关键决策：
  - 估价失败时是否允许直接提交（默认允许但提示无法预估）。

## 4. API 契约（如涉及）
- `POST /api/quote/estimate`
  - Request：`skuId?`、`quantity?`、`processPreference?`、`materialPreference?`、`precisionPreference?`、`leadTimePreference?`
  - Response：`currency`、`priceMin`、`priceMax`、`ruleVersion?`、`disclaimer`
- `POST /api/inquiries`
  - Request：`inquiryType`、`productId?`、`skuId?`、`name`、`email`、`phone?`、`company?`、`quantity?`、偏好字段、`notes?`、`attachments?`
  - Response：`id`、`status`、`createdAt`、`quoteEstimate`

## 5. 数据模型/存储（如涉及）
- `inquiry` 表持久化 request 与 estimate，保留规则版本信息（如有）。

## 6. 前端交互（如涉及）
- 页面/路由：`/quote`
- 必填规则：
  - service：必须上传至少 1 个附件
  - product：必须有 skuId

## 7. 安全与合规
- 上传限制必须前后端一致：扩展名、大小、MIME。
- 禁止在日志输出邮箱/电话/附件链接等敏感信息（最少化记录）。

## 8. 测试计划（必须覆盖边界）
- 手动回归路径：
  - service：不上传不可提交，上传后可提交
  - product：缺 skuId 不可提交，带参数可估价并提交
  - 接口失败：估价/提交失败有明确错误提示

## 9. 发布与监控
- 监控建议：询价提交成功率、估价失败率、上传失败率、/api 5xx 错误率。
