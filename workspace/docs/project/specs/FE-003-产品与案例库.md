# Tech Spec：FE-003 产品与案例库（列表+详情）

## 1. 目标与非目标
- 目标：完善 `/work` 列表与 `/work/:id` 详情，保证筛选/搜索/状态展示可用，并提供“产品询价入口”参数规范。
- 非目标：3D 预览、收藏、下载资料等增强能力。

## 2. 现状说明（基于代码）
- 相关入口：
  - 页面：`workspace/src/pages/WorkList.tsx`、`workspace/src/pages/WorkDetail.tsx`
  - 路由：`workspace/src/App.tsx`
  - API：`GET /api/products`、`GET /api/products/:id`
- 现有行为：列表与详情页面已存在（具体字段与交互以当前实现为准）。
- 现有问题：待补齐（筛选/搜索契约、空态/错误态、CTA 参数一致性）。

## 3. 设计概览
- 方案摘要：
  - 列表：优先实现关键字搜索与分页（或限制条数），再扩展多维筛选。
  - 详情：媒体与参数展示优先，SKU 展示与“立即询价”参数需与 FE-004 对齐。
- 关键决策：
  - 产品与案例是否合并为同一路由（当前合并为 `/work`），案例 API 后续补齐。

## 4. API 契约（如涉及）
- `GET /api/products`
  - 参数：`page`/`pageSize`/`q`/`featured`/（后续扩展 tags/material/process 等）
  - 返回：列表项字段（id/title/summary/cover/priceHint/tags...）
- `GET /api/products/:id`
  - 返回：详情字段 + skus + gallery

## 5. 前端交互（如涉及）
- 页面/路由：`/work`、`/work/:id`
- 询价入口参数：
  - 推荐：`/quote?type=product&productId=<id>&skuId=<id>`

## 6. 测试计划（必须覆盖边界）
- 手动回归路径：
  - `/work`：加载成功/失败/空态
  - `/work/:id`：不存在 id 的提示与回退
  - 详情 CTA：跳转到询价页并带参数

## 7. 发布与监控
- 发布步骤：前端静态资源发布；如涉及 API/DB 同步发布后端。
