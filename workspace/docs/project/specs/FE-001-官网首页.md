# Tech Spec：FE-001 官网首页

## 1. 目标与非目标
- 目标：实现 PRD 约定的官网首页展示与转化入口，并保证精选产品区块状态完整。
- 非目标：多语言体系、复杂动效与 3D 展示。

## 2. 现状说明（基于代码）
- 相关入口：
  - 页面：`workspace/src/pages/Home.tsx`
  - 路由：`workspace/src/App.tsx`
  - API：`GET /api/products?featured=true`
- 现有行为：已存在首页与精选产品区块。
- 现有问题：待补齐（例如空态/错误态体验、精选配置方式等）。

## 3. 设计概览
- 方案摘要：以 `Home.tsx` 为主，保持页面拆分与状态管理一致。
- 关键决策：精选展示来源（featured query / 后台配置）与展示数量（默认 3）。

## 4. API 契约（如涉及）
- 暂无新增 API；复用 `GET /api/products?featured=true`。

## 5. 数据模型/存储（如涉及）
- 暂无新增表；如需要“首页精选配置”，后续在 AD-004 站点设置中承载。

## 6. 前端交互（如涉及）
- 页面/路由：`/`
- 状态流转：
  - loading：显示骨架
  - success：展示最多 3 条
  - error：展示错误卡片 + 引导跳转 `/work`
  - empty：展示空态 + 引导跳转 `/work`

## 7. 安全与合规
- 无敏感数据；错误信息不泄露内部细节。

## 8. 测试计划（必须覆盖边界）
- 手动回归路径：
  - `/` 首屏 CTA、精选加载成功/失败/空态
  - 点击产品卡片进入 `/work/:id`

## 9. 发布与监控
- 发布步骤：前端静态资源发布。
- 监控建议：前端错误上报（如后续接入），首页白屏率与首屏性能指标（LCP）。
