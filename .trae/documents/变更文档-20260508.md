# 变更文档（2026-05-08）

目标：基于 `print-site-code` 项目，完成页面模式、后台会话保持、后台 CRUD 能力补齐、询价表单校验与文案优化、仪表盘跳转修复、页签图标/标题去 Trae 等需求，并完成构建验证。

## 1. 需求覆盖清单

### 1.1 功能新增
1. 页面模式：白天 / 黑夜 / 跟随系统
2. 后台管理登录态保持：已登录未登出时，前台↔后台切换不需要重新登录
3. 产品管理：新增产品增删改查（含 SKU 管理）
4. 报价规则：增删改查 + 启用规则
5. 询价管理：增删改查（含详情查看、备注编辑）
6. 仪表盘：卡片点击跳转到对应的产品/询价菜单

### 1.2 功能优化
1. 文案调整：
   - 代打服务：需要上传模型文件
   - 平台产品：选择现有产品，不需要上传文件
2. 代打服务询价表单新增输入校验（邮箱或电话至少其一、数量、附件等）
3. 后台登录：支持账号或邮箱 + 密码；输入框不默认展示账号/邮箱
4. 提交成功页：展示“订单编号”，不展示“线索编号”
5. 转化路径：优先引导浏览平台产品，弱化代打服务的注意力抢占

### 1.3 Bug 修复
1. 上传文件后提交询价无法点击：提交按钮受校验控制，上传完成后可正常提交；并补齐校验错误提示
2. 报价规则点击无响应：补齐后台报价规则编辑/删除与后端 PATCH/DELETE
3. 询价管理点进去无法查看详情：补齐询价详情页与路由
4. 后台产品管理无法点进去查看：补齐产品编辑页与路由
5. 上传文件后展示样式为文件项：上传区域展示为文件列表卡片
6. 浏览器页签图标不使用 Trae：替换 favicon.svg
7. 浏览器页签名称与页面匹配：默认 title 更新 + 主要页面动态 title

## 3. 实现说明

### 3.1 页面模式（白天/黑夜/跟随系统）
- 三档模式：`system | light | dark`，并持久化到 localStorage
- `system` 模式会监听 `prefers-color-scheme` 变化并实时切换
- 统一在 `<html>` 上切换 `dark` class（Tailwind `darkMode: "class"`）
- 入口：
  - 前台导航：右上角模式按钮
  - 后台侧栏：模式按钮
- 相关实现：
  - [useTheme.ts](file:///workspace/print-site-code/workspace/src/hooks/useTheme.ts)
  - [ThemeToggle.tsx](file:///workspace/print-site-code/workspace/src/components/ThemeToggle.tsx)
  - [App.tsx](file:///workspace/print-site-code/workspace/src/App.tsx)
  - [SiteShell.tsx](file:///workspace/print-site-code/workspace/src/components/SiteShell.tsx)
  - [AdminLayout.tsx](file:///workspace/print-site-code/workspace/src/pages/admin/AdminLayout.tsx)

### 3.2 后台会话保持与登录方式优化
- 前台“后台”入口改为 `/admin`（受保护路由），避免仅跳转登录页导致的体验割裂
- 登录页：若已有 token，会先调用 `/api/auth/me` 校验，通过则直接进入仪表盘
- 路由守卫：仅在 `401` 才清理 token，减少临时网络错误导致的被动登出
- 后台登录支持：账号或邮箱 + 密码（后端 identifier 兼容）
- 相关实现：
  - 前端：
    - [Login.tsx](file:///workspace/print-site-code/workspace/src/pages/admin/Login.tsx)
    - [RequireAdmin.tsx](file:///workspace/print-site-code/workspace/src/pages/admin/RequireAdmin.tsx)
    - [auth.ts](file:///workspace/print-site-code/workspace/src/stores/auth.ts)
  - 后端：
    - [auth.ts (routes)](file:///workspace/print-site-code/workspace/api/routes/auth.ts)
