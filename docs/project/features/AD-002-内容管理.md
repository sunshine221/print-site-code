## 背景与目标
- 背景：公开端展示依赖可维护的内容（产品、SKU、案例、素材），需要后台提供 CRUD 与发布控制。
- 目标用户/使用场景：管理员在后台新增/编辑/上下架内容，调整精选与排序，维护媒体素材并复用。

## 范围
- In scope：
  - 产品管理：产品 CRUD、上下架、精选、排序、标签、参数字段、SEO 字段。
  - SKU 管理：SKU 规格/属性、参考价区间、库存（可选）、上下架。
  - 案例管理：案例 CRUD、置顶、关联产品、媒体与文案。
  - 素材库：图片/视频/3D 文件/附件的上传与管理、删除保护（被引用提示）。
- Out of scope：
  - 多人协作与复杂权限体系。
  - 批量导入/导出（可选增强）。

## 用户故事
- 作为管理员，我想在后台维护产品与 SKU，并上传素材复用到产品/案例，从而保证公开端内容持续更新且一致。

## 验收标准（必须可测试）
- [ ] 管理员登录后可进入内容管理页面（产品/素材等），未登录不可访问。
- [ ] 产品支持新增、编辑、删除、上下架；公开端只展示已发布内容。
- [ ] SKU 可在产品下维护，支持至少：skuCode、attributes、priceMin/priceMax、published。
- [ ] 素材上传后可在产品/案例中引用；删除已被引用的素材会提示并阻止或要求确认（策略在 Tech Spec 定稿）。
- [ ] 任一 CRUD 失败时给出可理解错误提示，不产生半成品脏数据。

## 边界与反例用例
- 并发编辑：同一产品被同时编辑时的最后写入策略（先按“最后写入覆盖”，后续可增强为乐观锁）。
- 字段校验：标题/summary 为空；skuCode 重复；价格区间不合法（min > max）。
- 删除保护：被引用的媒体删除策略与提示一致。

## 依赖与约束
- 相关入口：
  - 前端页面：`src/pages/admin/Products.tsx`、`src/pages/admin/Media.tsx`（如后续新增）
  - 后端路由：`api/routes/admin/products.ts`、`api/routes/admin/media.ts`
  - 数据表：`product`、`product_sku`、`media`、`product_media`、`case_study` 等（见技术架构 DDL）

## 实现拆分（最小可回滚）
- 子任务 1：产品 CRUD（基础字段 + published/featured）。
- 子任务 2：SKU 维护（列表/新增/编辑/删除 + published）。
- 子任务 3：素材库（上传/引用/删除保护）。
- 子任务 4：案例管理与关联（可后置到 P1/P2）。

## 验证方式
- 自动化：`npm run check`、`npm run lint`、`npm run build`
- 手动路径：
  - 登录后台 → 新增产品 → 上架 → 前台 `/work` 可见
  - 编辑产品/sku 后刷新前台详情页确认生效
  - 上传素材并引用到产品/案例（如已实现）

## 发布与回滚
- 发布检查点：CRUD 正常、发布控制生效、删除保护不误删。
- 回滚方式：若涉及数据库迁移，发布前备份 SQLite；回滚时恢复备份并回滚后端版本。
