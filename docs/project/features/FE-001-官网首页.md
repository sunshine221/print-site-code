# 需求卡片：FE-001 官网首页

## 基本信息
- 编号：FE-001
- 名称：官网首页
- 状态：draft
- 关联文档：
  - PRD：`.trae/documents/3D打印产品展示与管理平台-PRD.md`

## 背景与目标
- 背景：需要一个对外展示首页，承载品牌主视觉、能力概览、精选内容与转化入口。
- 目标用户/使用场景：访客进入网站后第一屏快速理解能力与路径，转化到询价或浏览产品。
- 业务目标（可量化）：提升进入 `/quote` 或 `/work` 的点击率；降低首页跳出率（指标在运营阶段补齐）。

## 范围
- In scope：
  - 顶部导航：Logo、主导航（产品/案例、能力工艺、询价联系）、CTA（可选）、语言切换（可选）。
  - 首屏主视觉：主标题 + 一句话价值主张 + 关键能力标签（尺寸/精度/交期/材料）+ 引导动作。
  - 能力与工艺速览：工艺类型卡片（FDM/SLA/SLS/金属等可配置）、材料与后处理、质量保障说明。
  - 精选展示：置顶产品/案例卡片，支持点击进入详情；支持手动配置展示顺序（先做“精选产品”，案例可后续补）。
  - 转化区块：明显的行动入口（询价/联系），包含承诺信息（响应时间/隐私说明）。
- Out of scope：
  - 多语言完整体系（可先留接口/入口）。
  - 复杂动效与 3D 展示（可后续增强）。

## 用户故事
- 作为访客，我想在首页快速了解可提供的 3D 打印能力，从而决定是直接询价还是先浏览产品。

## 验收标准（必须可测试）
- [ ] 访问 `/` 可正常渲染首屏主视觉、精选展示与 CTA，不出现空白页。
- [ ] 首屏 CTA 可跳转到 `/quote?type=service` 与 `/work`（或与当前实现一致的路径）。
- [ ] 精选产品区块在加载中、成功、失败三种状态下均有明确展示。
- [ ] 精选产品点击可跳转到对应详情页（`/work/:id`）。
- [ ] 页面在移动端与桌面端均可用，内容不溢出、不遮挡 CTA。

## 边界与反例用例
- 空状态：精选产品为空时显示“暂无精选/建议浏览全部产品”的引导。
- 异常：`/api/products?featured=true` 请求失败时显示可理解错误提示，不影响页面其他区域可用。
- 性能：图片列表支持懒加载或限制展示数量（默认最多 3 条）。

## 依赖与约束
- 相关入口：
  - 页面：`src/pages/Home.tsx`
  - 路由：`src/App.tsx`
  - API：`GET /api/products?featured=true`

## 实现拆分（最小可回滚）
- 子任务 1：首页首屏与 CTA（静态内容为主）。
- 子任务 2：精选产品（含 loading/error/empty 状态）。
- 子任务 3：导航与站点壳（如需补齐导航项与高亮逻辑）。

## 验证方式
- 自动化：`npm run check`、`npm run lint`、`npm run build`
- 手动路径：
  - 访问 `/`，检查首屏 CTA、精选产品、错误态与空态
  - 点击“浏览平台产品”进入 `/work`
  - 点击产品卡片进入 `/work/:id`

## 发布与回滚
- 发布检查点：首页首屏渲染、CTA 跳转、精选产品加载成功/失败。
- 回滚方式：仅涉及前端页面与组件变更，可直接回滚到上一版本静态资源。
