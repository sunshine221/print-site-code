## 背景与目标
- 背景：询价是核心转化闭环，需要覆盖“代打服务（上传文件）”与“平台产品（选 SKU，无需上传）”两条路径，并产出可跟进的线索。
- 目标用户/使用场景：访客在公开端提交询价；运营/管理员在后台跟进处理。
- 业务目标（可量化）：提高询价提交成功率，缩短管理员首次响应时间（指标在运营阶段补齐）。

## 范围
- In scope：
  - 双路径入口：
    - 代打服务：`/quote?type=service`（可选上传附件）
    - 平台产品：`/quote?type=product&productId=...&skuId=...`（无需上传附件）
  - 表单字段（精简）：姓名/电话、SKU（仅产品路径）、数量、备注。
  - 价格展示：产品路径显示 “SKU 价格区间 × 数量” 的价格总结；代打路径固定“价格面议”。
  - 提交询价：调用 `POST /api/inquiries` 创建线索，返回线索编号与状态。
  - 提交成功回执：展示线索编号、预计响应说明（文案可后续配置）。
- Out of scope：
  - 邮件回执（可选增强）。
  - 复杂的模型体积/尺寸自动解析（可后续扩展到模型指标）。

## 用户故事
- 作为访客，我想通过上传文件或选择 SKU 提交询价，并得到一个可追踪编号，从而知道已提交成功并等待响应。

## 验收标准（必须可测试）
- [ ] 访问 `/quote?type=service`，不上传附件也可以提交（价格面议）。
- [ ] 访问 `/quote?type=product&productId=...`，在缺少 `skuId` 时给出明确提示且无法提交。
- [ ] 访问 `/quote?type=product&productId=...&skuId=...`，展示“价格总结”（sku 有价格则按数量计算区间；无价格则显示“价格面议”）。
- [ ] 点击“提交询价”会调用 `POST /api/inquiries`，成功后展示线索 `id` 与成功提示。
- [ ] 表单存在 loading/error 状态：接口失败时给出可理解错误信息，不白屏。

## 边界与反例用例
- 输入校验：电话为空/格式不合法；quantity <= 0；超长备注；特殊字符。
- 上传：0 个文件、超大文件、非法扩展名、上传中断重试（前后端限制一致）。
- 反复点击：提交按钮需要防重复提交（前端 submitting 状态 + 后端幂等策略后续补齐）。
