# Tech Spec：AD-001 管理员登录与鉴权

## 1. 目标与非目标
- 目标：后台路由与后台 API 全面鉴权；提供稳定的登录态管理与 401 退登机制。
- 非目标：多角色权限体系、忘记密码。

## 2. 现状说明（基于代码）
- 相关入口：
  - 前端：`src/pages/admin/Login.tsx`、`src/pages/admin/RequireAdmin.tsx`、`src/stores/auth.ts`
  - 后端：`api/routes/auth.ts`、`api/auth.ts`
  - 环境变量：`.env.example`（JWT_SECRET、ADMIN_EMAIL、ADMIN_USERNAME、ADMIN_PASSWORD）
- 现有行为：已存在登录接口与后台路由保护。
- 现有问题：待补齐（错误码统一、token 过期策略、请求层统一注入 token）。

## 3. 设计概览
- token 存储：localStorage（key 统一），同时在 store 中维护 user。
- 请求注入：`apiFetch` 统一加 Authorization: Bearer <token>（若项目已实现则确认一致性）。
- 401 处理：前端收到 401 → clear auth → navigate /admin/login。

## 4. API 契约
- `POST /api/auth/login`
  - Request：`username`、`password`
  - Response：`token`、`user { id,email,name }`
  - Error：400 缺参数；401 无效凭证
- `GET /api/auth/me`
  - Header：Authorization
  - Response：`user { id,email,name }`
  - Error：401 未授权

## 5. 安全
- 不输出 password/token 到日志与错误信息。
- JWT_SECRET 仅通过环境变量配置，不写入仓库。

## 6. 测试计划
- 手动回归路径：
  - 未登录访问 `/admin` 跳转 `/admin/login`
  - 登录成功 → dashboard
  - token 篡改/过期 → 401 → 退登

## 7. 发布与回滚
- 发布：前后端同步发布；必要时迁移管理员表数据。
- 回滚：回滚后端版本与前端静态资源；如 DB 结构变化需恢复备份。
