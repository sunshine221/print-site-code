# Tech Spec：FE-004 询价/联系（双路径）

## 1. 目标与非目标
- 目标：双路径询价（service/product）可用，表单精简为姓名+电话+数量+备注（product 额外要求 sku），并可为后台线索管理提供数据闭环。
- 目标：移除估价能力（不再提供 `POST /api/quote/estimate`），价格仅展示“产品价格总结”或“价格面议”。
- 非目标：邮件回执、复杂幂等键体系、代打服务精确报价/自动模型解析（可后续增强）。

## 2. 现状说明（基于代码）
- 相关入口：
  - 页面：`src/pages/Quote.tsx`
  - API：`POST /api/quote/estimate`、`POST /api/inquiries`
- 现有行为：页面已支持双路径参数 `type`，并实现估价与提交；后端 `POST /api/inquiries` 会写入 `estimate_json` 并返回 `quoteEstimate`。
- 现有问题：表单字段过多；前后端必填不一致（后端强制 email）；service 目前强制上传附件。

## 3. 设计概览
- 方案摘要：
  - 双路径参数：`type=service|product`，product 需 `productId` 与 `skuId`。
  - 价格展示：
    - product：前端基于 `GET /api/products/:id` 获取 sku 的 `priceMin/priceMax/currency` 并乘以数量生成“价格总结”；若 sku 无价格则展示“价格面议”。
    - service：固定展示“价格面议”。
  - 提交：
    - service：仅允许上传附件（可选）；不上传也允许提交。
    - product：必须携带 `skuId`；不允许上传附件。
- 关键决策：
  - 直接移除 `POST /api/quote/estimate` 端点与前端调用；不再落库/返回 `quoteEstimate`。

## 4. API 契约（如涉及）
- `POST /api/inquiries`
  - Request：
    - 通用：`inquiryType`（`service_print|platform_product`）、`name`、`phone`、`quantity`、`notes?`
    - product：`productId?`、`skuId`（必填）
    - service：`attachments?`（可选，仅该类型允许；数组元素包含 `url/name/size?/mimeType?`）
  - Response：`id`、`status`、`createdAt`
  - Error（保持现有结构 `success:false,error`）：
    - 缺必填：`name|phone|quantity` 为空或非法
    - 类型非法：`inquiryType` 非法
    - product：缺 `skuId` / `skuId` 不存在 / `skuId` 不属于 `productId`（当传了 `productId` 时）
    - service：`attachments` 内容非法（如给了但 `url` 为空时）

## 5. 数据模型/存储（如涉及）
- `inquiry` 表继续持久化 request（`request_json`）与附件关系（如有）。
- 不再生成/写入 `estimate_json` 与 `pricing_rule_version`；历史数据保留，后台展示兼容。

## 6. 前端交互（如涉及）
- 页面/路由：`/quote`
- 必填规则：
  - 通用：姓名、电话、数量（>=1）
  - product：必须有 `skuId`（同时需要能从 `GET /api/products/:id` 中解析到对应 sku）
  - service：附件可选
- 无效参数处理：
  - product：当 `productId/skuId` 无效或不匹配时，展示错误提示并引导返回 `/work`。

## 7. 安全与合规
- 上传限制必须前后端一致：扩展名、大小、MIME。
- 禁止在日志输出邮箱/电话/附件链接等敏感信息（最少化记录）。

## 8. 测试计划（必须覆盖边界）
- 手动回归路径：
  - service：不上传也可提交；上传可选；提交成功回执展示询价 id
  - product：缺 skuId 不可提交；skuId 无效/不属于 productId 时提示并引导回 `/work`；sku 有价格时展示价格总结，无价格则“价格面议”
  - 接口失败：提交失败有明确错误提示

## 9. 发布与监控
- 监控建议：询价提交成功率、上传失败率、/api 5xx 错误率。
