# Tech Spec：AD-002 内容管理（产品/SKU/案例/素材）

## 1. 目标与非目标
- 目标：后台提供产品/SKU/素材（以及可选案例）的可用 CRUD，并与公开端展示形成一致闭环。
- 非目标：复杂权限、多端协作、批量导入导出。

## 2. 现状说明（基于代码）
- 相关入口：
  - 前端：`src/pages/admin/Products.tsx`
  - 后端：`api/routes/admin/products.ts`、`api/routes/admin/media.ts`
  - 数据：SQLite（迁移见 `migrations`）
- 现有行为：相关页面与路由已存在（具体功能以当前实现为准）。
- 现有问题：待补齐（字段约束、删除保护、案例管理接入等）。

## 3. 设计概览
- 产品与 SKU：
  - 产品为聚合根，SKU 作为子资源。
  - published 控制公开端可见性；featured 控制首页精选。
- 素材：
  - 上传到 COS（或本地）后落库 media 记录。
  - 引用关系通过关联表维护，删除策略与提示需要明确。

## 4. API 契约（如涉及）
- admin 产品：
  - `POST /api/admin/products`
  - `PATCH /api/admin/products/:id`
  - `DELETE /api/admin/products/:id`
- admin 素材：
  - `POST /api/admin/media/presign`
  - `POST /api/admin/media`
  - `DELETE /api/admin/media/:id`

## 5. 数据模型/存储（如涉及）
- 参考技术架构 DDL：product、product_sku、media、product_media、case_study 等。
- skuCode 唯一性与价格区间校验策略需要落到后端。

## 6. 测试计划
- 手动回归路径：
  - 新增产品 → 上架 → 前台 `/work` 可见
  - 新增 SKU → 详情页可见 → 询价参数可用
  - 上传素材 → 引用 → 删除保护策略符合预期

## 7. 发布与回滚
- 发布：如有迁移，先备份 SQLite；再发布后端与前端。
- 回滚：恢复备份并回滚后端版本；前端回滚静态资源。
