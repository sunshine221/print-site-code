# Tech Spec：AD-003 询价线索管理

## 1. 目标与非目标
- 目标：后台可查看/更新询价线索，支持状态流转与内部备注，形成“提交→跟进→关闭”的闭环。
- 非目标：多人协作、复杂跟进记录系统。

## 2. 现状说明（基于代码）
- 相关入口：
  - 前端：`src/pages/admin/Inquiries.tsx`
  - 后端：`api/routes/admin/inquiries.ts`
  - 数据表：`inquiry`、`inquiry_attachment`、`media`
- 现有行为：页面与路由已存在（具体字段以当前实现为准）。
- 现有问题：待补齐（筛选/分页、导出、附件可访问性策略）。

## 3. 设计概览
- 状态枚举：`new | processing | quoted | closed`
- 更新策略：PATCH 更新 status 与 internalNote，后端校验枚举与权限。
- 列表策略：默认按 createdAt 倒序，分页参数后续补齐。

## 4. API 契约（如涉及）
- `GET /api/admin/inquiries`
- `GET /api/admin/inquiries/:id`
- `PATCH /api/admin/inquiries/:id`（更新 status/internalNote）
- `GET /api/admin/inquiries/export`（可选）

## 5. 测试计划
- 手动回归路径：
  - 公开端提交询价 → 后台可见
  - 更新状态与备注 → 刷新后保持
  - 非法状态 → 后端拒绝并提示

## 6. 发布与回滚
- 发布：后端 API 与前端页面同步发布。
- 回滚：如无迁移，回滚后端版本即可；如有迁移，恢复 SQLite 备份。
